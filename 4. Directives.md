# Angular Directives

A **directive** is a class that **modifies the behavior or appearance of DOM elements** in an Angular application.

Directives allow developers to **add functionality to HTML elements without creating a new component**.

Key points:

* Used to **change behavior or appearance of DOM elements**
* Implemented using the `@Directive` decorator
* Can implement **Angular lifecycle hooks**
* **Do not have their own template** (except components)

In Angular, **components are technically a type of directive** with a template.

---

# Types of Directives

Angular provides three main types of directives.

## 1. Structural Directives

Structural directives **change the structure of the DOM** by **adding or removing elements**.

They usually start with `*`.

Examples:

| Directive   | Purpose                                       |
| ----------- | --------------------------------------------- |
| `*ngIf`     | Conditionally displays an element             |
| `*ngFor`    | Repeats an element for each item in a list    |
| `*ngSwitch` | Displays elements based on a switch condition |

Modern Angular versions also support **new control flow syntax**:

| Old Syntax  | New Syntax   |
| ----------- | ------------ |
| `*ngIf`     | `@if {}`     |
| `*ngFor`    | `@for {}`    |
| `*ngSwitch` | `@switch {}` |

Example:

```html
@if (isLoggedIn) {
  <p>Welcome User</p>
}
```

Example of looping:

```html
@for (item of items; track item.id) {
  <p>{{ item }}</p>
}
```

Example of switch:

```html
@switch (role) {
  @case ('admin') {
    <p>Admin Panel</p>
  }
  @default {
    <p>User Panel</p>
  }
}
```

Structural directives **modify DOM layout**, which may affect performance if used excessively.

# Built-in Attribute Directives

Attribute directives **modify the behavior or appearance of existing elements** without changing the DOM structure.

They must be added to the component’s `imports` array.

---

# NgClass

`NgClass` **adds or removes CSS classes dynamically**.

Example:

```ts
import { NgClass } from '@angular/common';

@Component({
  template: `<div [ngClass]="isSpecial ? 'special' : ''">
    This div is special
  </div>`,
  imports: [NgClass]
})
export class AppComponent {
  isSpecial = true;
}
```

---

# NgStyle

`NgStyle` **dynamically applies CSS styles** to elements.

Example:

```ts
import { NgStyle } from '@angular/common';

@Component({
  imports: [NgStyle]
})
export class AppComponent {

  currentStyles: Record<string, string> = {};

  setCurrentStyles() {
    this.currentStyles = {
      'font-style': this.canSave ? 'italic' : 'normal',
      'font-weight': !this.isUnchanged ? 'bold' : 'normal',
      'font-size': this.isSpecial ? '24px' : '12px'
    };
  }

}
```

Template:

```html
<div [ngStyle]="currentStyles">
  Styled content
</div>
```

---

# NgModel

`NgModel` provides **two-way data binding** between form inputs and component data.

Example:

```html
<input [(ngModel)]="username">
<p>{{ username }}</p>
```

This synchronizes:

```
UI ↔ Component Data
```

---

# Hosting a Directive Without a DOM Element

Angular provides `<ng-container>`.

`ng-container` groups elements **without creating an actual DOM element**.

Example:

```html
<ng-container *ngIf="isVisible">
  <p>Visible content</p>
</ng-container>
```

This keeps the DOM **cleaner and avoids layout issues**.

# Creating a Custom Attribute Directive

To create a directive:

```
ng generate directive highlight
```

Directives commonly use **ElementRef** to access the host element.

Example:

```ts
import { Directive, ElementRef, inject } from '@angular/core';

@Directive({
  selector: '[appHighlight]'
})
export class HighlightDirective {

  private el = inject(ElementRef);

  constructor() {
    this.el.nativeElement.style.backgroundColor = 'yellow';
  }

}
```

Usage:

```html
<p appHighlight>Highlight me!</p>
```

---

# Handling User Events in Directives

Directives can respond to events using the `host` property.

Example:

```ts
import { Directive, ElementRef, inject } from '@angular/core';

@Directive({
  selector: '[appHighlight]',
  host: {
    '(mouseenter)': 'onMouseEnter()',
    '(mouseleave)': 'onMouseLeave()'
  }
})
export class HighlightDirective {

  private el = inject(ElementRef);

  onMouseEnter() {
    this.highlight('yellow');
  }

  onMouseLeave() {
    this.highlight('');
  }

  private highlight(color: string) {
    this.el.nativeElement.style.backgroundColor = color;
  }

}
```

Usage:

```html
<p appHighlight>Hover over me!</p>
```

When the user **moves the mouse over the element**, the background color changes.
