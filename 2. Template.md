# Angular Templates

Templates in Angular define the **user interface (UI) of a component**.
They use **HTML combined with Angular syntax** to display data and respond to user actions.

---

# 1. Interpolation

Interpolation displays **component data inside HTML** using double curly braces.

Syntax:

```html
{{ expression }}
```

Example:

```html
<h2>{{ name }}</h2>
```

Component:

```ts
name = "Urmi";
```

Output:

```
Urmi
```

Interpolation can also perform simple expressions:

```html
{{ 5 + 5 }}
{{ user.age }}
```

---

# 2. Reference Variables

Reference variables allow you to **reference an element inside the template**.

Syntax:

```
#variableName
```

Example:

```html
<input #username>
<button (click)="show(username.value)">Submit</button>
```

Component:

```ts
show(value: string) {
  console.log(value);
}
```

---

# 3. Null-Safe Navigation (?.)

Prevents errors when accessing **null or undefined values**.

Example:

```html
<p>{{ user?.name }}</p>
```

If `user` is `null`, Angular will **not throw an error**.

Without safe navigation:

```html
{{ user.name }}
```

This would cause an **error if user is null**.

---

# 4. Structural Directives

Structural directives **change the DOM layout**.

Common ones:

### ngIf

Displays an element **only if condition is true**.

```html
<p *ngIf="isLoggedIn">Welcome</p>
```

---

### ngFor

Used to **loop through lists**.

```html
<li *ngFor="let item of items">
  {{ item }}
</li>
```

---

### ngSwitch

Used for **multiple conditions**.

```html
<div [ngSwitch]="role">
  <p *ngSwitchCase="'admin'">Admin User</p>
  <p *ngSwitchDefault>User</p>
</div>
```

---

# 5. ngTemplateOutlet

Used to **reuse a template multiple times**.

Example:

```html
<ng-template #greet>
  <p>Hello Angular</p>
</ng-template>

<div *ngTemplateOutlet="greet"></div>
```

Useful when **repeating template structures**.

---

# 6. Statements and $event

Angular templates can **execute statements when events occur**.

Example:

```html
<button (click)="increment()">Click</button>
```

Event object can be accessed using `$event`.

```html
<input (input)="onInput($event)">
```

Component:

```ts
onInput(event: any) {
  console.log(event.target.value);
}
```

---

# 7. Alias

Alias allows **renaming variables in templates**.

Example in `ngIf`:

```html
<div *ngIf="user as u">
  {{ u.name }}
</div>
```

Here:

```
u → alias for user
```

---

# 8. Pipes ( | )

Pipes transform data before displaying it.

Example:

```html
<p>{{ name | uppercase }}</p>
<p>{{ today | date }}</p>
```

Common built-in pipes:

* `date`
* `uppercase`
* `lowercase`
* `currency`
* `percent`
* `json`

Example:

```html
{{ price | currency }}
```

---

# 9. Attribute Binding

Used to bind **HTML attributes dynamically**.

Syntax:

```html
[attr.attributeName]="value"
```

Example:

```html
<button [attr.disabled]="isDisabled">Submit</button>
```

Another example:

```html
<img [src]="imageUrl">
```

---

# 10. TrackBy

Used with `ngFor` to **improve performance** when rendering lists.

Without TrackBy → Angular re-renders entire list.

With TrackBy → Angular **updates only changed elements**.

Example:

```html
<li *ngFor="let user of users; trackBy: trackById">
  {{ user.name }}
</li>
```

Component:

```ts
trackById(index: number, user: any) {
  return user.id;
}
```

---

# Why TrackBy is Important

Benefits:

* Improves **performance**
* Reduces **DOM updates**
* Useful for **large lists**
