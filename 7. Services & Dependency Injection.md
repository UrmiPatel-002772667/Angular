# Angular Services

A **Service** in Angular is a **self-contained, reusable class that performs a specific task** and can be shared across multiple components.

Services help separate **business logic, data access, and shared functionality** from UI components.

This improves **code reusability, maintainability, and scalability**.

## When to Use Services

### 1. Sharing Data Between Components

Services allow multiple components to **access the same data source**.

Example Component:

```ts
dummyData: any;
constructor(private shared_Data: SharedData) {
  this.dummyData = this.shared_Data.userData;
}
```

Template:

```html
<ul>
  <li>Id: {{dummyData.id}}</li>
  <li>Name: {{dummyData.name}}</li>
  <li>UserName: {{dummyData.username}}</li>
  <li>Email: {{dummyData.email}}</li>
</ul>
```

### 2. Sharing Business Logic Across Components

Services are commonly used to **centralize business logic** that multiple components can reuse.

Example:

```ts
isEligible: boolean;
constructor(private shared_Data: SharedData) {
  this.isEligible = this.shared_Data.isEligibleForSubscription();
}
```

Template:

```html
@if(isEligible){
  <h4 class="text-success">
    {{dummyData.name}} is Eligible for Premium Subscription.
  </h4>
}
@else{
  <h4 class="text-danger">
    {{dummyData.name}} is Not Eligible for Premium Subscription.
  </h4>
}
```

---

# 3. Interacting with APIs or External Data Sources

Services are commonly used to **handle HTTP requests**.

Example Component:

```ts
apiData: any;

ngOnInit(): void {
  this.getData();
}

getData() {
  this.shared_Data.getUserData().subscribe(res => {
    this.apiData = res;
  });
}
```

Template:

```html
@for(data of apiData; track data.id){
  <ul>
    <li>Id: {{data.id}}</li>
    <li>Name: {{data.name}}</li>
    <li>UserName: {{data.username}}</li>
    <li>Email: {{data.email}}</li>
  </ul>
}
```

## Creating a Service

Generate a service using Angular CLI:

```bash
ng generate service shared-data
```

Example Service:

```ts
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class SharedData {
  userData = {
    id: 1,
    name: "Urmi",
    username: "urmi01",
    email: "urmi@email.com"
  };

  isEligibleForSubscription() {
    return true;
  }

}
```

`providedIn: 'root'` makes the service **available throughout the application**.

---

### Purpose of Using Services

**1. Code Organization** Keeps components clean by moving business logic into services.  
**2. Code Reusability** The same service can be used by **multiple components**.  
**3. Separation of Concerns** UI logic stays in components while **data logic stays in services**.  
**4. Easier Testing** Services can be **tested independently**.  

# Dependency Injection (DI)

**Dependency Injection** is a design pattern where **a class receives its dependencies from an external source rather than creating them itself**.

Angular has a **built-in Dependency Injection system**.

Instead of creating objects manually, Angular **injects them automatically**.

Example:

```ts
constructor(private shared_Data: SharedData) {}
```

Angular automatically provides the **SharedData service instance**.

---

## How Dependency Injection Works

### 1. Create a Service

```ts
@Injectable({
  providedIn: 'root'
})
export class SharedData {}
```

---

### 2. Register the Service

Angular registers services using:

* `@Injectable`
* `providers` array
* `providedIn: 'root'`

Example:

```ts
@Injectable({
  providedIn: 'root'
})
```

---

### 3. Inject the Service into Components

```ts
constructor(private sharedData: SharedData) {}
```

Angular automatically creates and injects the instance.

---

## Providers in Angular

Providers tell Angular **how to create a dependency**.

Types of providers:

| Provider Type     | Description                              |
| ----------------- | ---------------------------------------- |
| Class Provider    | Uses a class to create service           |
| Value Provider    | Provides a fixed value                   |
| Factory Provider  | Creates service using a factory function |
| Existing Provider | Reuses another service                   |

Example:

```ts
providers: [SharedData]
```

---

### Benefits of Dependency Injection

* Reduces tight coupling
* Improves code modularity
* Makes testing easier
* Improves maintainability
* Promotes reusable components
