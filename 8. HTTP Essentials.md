# HTTP Essentials (Angular)

HTTP in Angular is used to **communicate with backend servers or APIs** to fetch, send, update, or delete data.

Angular provides the **HttpClient module** to make HTTP requests.

It supports operations such as:

* GET – retrieve data
* POST – send new data
* PUT – update existing data
* DELETE – remove data

These operations allow Angular applications to **interact with databases and external APIs**.

---

# HttpClient Module

To use HTTP services in Angular, import **HttpClientModule**.

Example:

```ts
import { HttpClientModule } from '@angular/common/http';

@NgModule({
  imports: [HttpClientModule]
})
export class AppModule {}
```

After importing, Angular applications can make HTTP requests using **HttpClient**.

---

# Injecting HttpClient

The HttpClient service is injected into components or services using **Dependency Injection**.

Example:

```ts
import { HttpClient } from '@angular/common/http';

constructor(private http: HttpClient) {}
```

---

# Making HTTP Requests

## 1. GET Request

Used to **retrieve data from a server**.

Example:

```ts
getUsers() {
  return this.http.get('https://jsonplaceholder.typicode.com/users');
}
```

Usage in component:

```ts
this.service.getUsers().subscribe(data => {
  console.log(data);
});
```

---

# 2. POST Request

Used to **send data to the server**.

Example:

```ts
addUser(user: any) {
  return this.http.post('https://api.example.com/users', user);
}
```

---

# 3. PUT Request

Used to **update existing data**.

Example:

```ts
updateUser(id: number, user: any) {
  return this.http.put(`https://api.example.com/users/${id}`, user);
}
```

---

# 4. DELETE Request

Used to **delete data from the server**.

Example:

```ts
deleteUser(id: number) {
  return this.http.delete(`https://api.example.com/users/${id}`);
}
```

---

# Observables in HTTP

Angular HTTP methods return **Observables**.

To get the response, we must **subscribe()**.

Example:

```ts
this.http.get(url).subscribe(response => {
  console.log(response);
});
```

Why `subscribe()` is used:

* Executes the HTTP request
* Receives response data
* Handles errors
* Handles asynchronous operations

---

# Handling Errors

Errors can be handled using `catchError` or inside `subscribe`.

Example:

```ts
this.http.get(url).subscribe({
  next: data => console.log(data),
  error: err => console.error(err)
});
```

---

# HTTP Headers

Headers send **additional information with HTTP requests**.

Example:

```ts
import { HttpHeaders } from '@angular/common/http';

const headers = new HttpHeaders({
  'Content-Type': 'application/json'
});

this.http.get(url, { headers });
```

---

# HTTP Parameters

Used to send **query parameters** in requests.

Example:

```ts
import { HttpParams } from '@angular/common/http';

const params = new HttpParams()
  .set('page', '1')
  .set('limit', '10');

this.http.get(url, { params });
```

Example URL generated:

```
api/users?page=1&limit=10
```

---

# HTTP Interceptors

Interceptors allow developers to **intercept and modify HTTP requests or responses**.

Common uses:

* Add authentication tokens
* Log requests
* Handle errors globally

Example:

```ts
intercept(req: HttpRequest<any>, next: HttpHandler) {
  const modifiedReq = req.clone({
    setHeaders: {
      Authorization: 'Bearer token'
    }
  });

  return next.handle(modifiedReq);
}
```

---

# Best Practice: Use Services for HTTP Calls

HTTP requests should be placed inside **services**, not components.

Example Service:

```ts
@Injectable({
  providedIn: 'root'
})
export class UserService {

  constructor(private http: HttpClient) {}

  getUsers() {
    return this.http.get('https://api.example.com/users');
  }

}
```

Component:

```ts
this.userService.getUsers().subscribe(data => {
  this.users = data;
});
```

# Summary

HTTP Essentials in Angular include:

* **HttpClientModule** for HTTP communication
* HTTP methods: **GET, POST, PUT, DELETE**
* Observables and `subscribe()`
* HTTP headers and parameters
* Error handling
* HTTP interceptors

Using these features, Angular applications can **communicate with backend APIs efficiently and manage data dynamically**.
