# Angular Forms

Angular Forms are a **module in Angular used to handle user input, manage form state, and validate form data**.

They provide a **structured and scalable way to manage forms** in Angular applications.

Angular offers two main approaches for handling forms:

1. **Template-Driven Forms**
2. **Reactive Forms**

---

### When to Use Angular Forms

Angular forms are useful when:

* Complex form validations are required
* Large-scale applications are being developed
* Dynamic form generation is needed
* Two-way data binding is required
* Form state and validation must be controlled

---

## 1. Template-Driven Forms

Template-driven forms are **created using HTML templates** and rely heavily on Angular directives.

They are **best suited for simple forms**.

To use template-driven forms, import: `FormsModule`  

from `@angular/forms`.

### Steps to Create a Template-Driven Form

1. Import `FormsModule`
2. Create an HTML form
3. Use `ngForm` directive
4. Use `ngModel` for form controls
5. Handle submission using `ngSubmit`

Example:

```html
<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">

  <input
    type="text"
    name="username"
    ngModel
    required
  >

  <button type="submit">Submit</button>

</form>
```

Component:

```ts
onSubmit(form: any) {
  console.log(form.value);
}
```

---

#### Built-In Validators

Angular provides several built-in validation rules.

| Validator   | Purpose                          |
| ----------- | -------------------------------- |
| `required`  | Ensures the field has a value    |
| `minlength` | Minimum number of characters     |
| `maxlength` | Maximum number of characters     |
| `pattern`   | Matches a specific regex pattern |
| `email`     | Validates email format           |

Example:

```html
<input name="email" ngModel required email >
```

---

### novalidate Attribute

`novalidate` disables **default browser validation**.

It is useful when:

* Custom Angular validation is used
* Server-side validation is preferred
* Browser validation messages should be avoided

Example:

```html
<form novalidate>
```

# Advantages of Template-Driven Forms

* Easy to implement
* Minimal setup required
* Ideal for small applications
* Supports two-way data binding
* Quick prototyping

# Disadvantages of Template-Driven Forms

* Limited control over form structure
* Difficult debugging in large forms
* Not suitable for complex applications
* Limited scalability
* Harder to unit test

---

## 2. Reactive Forms

Reactive forms are **created programmatically using TypeScript** instead of relying mainly on HTML.

They provide **more control, better scalability, and improved testability**.

To use reactive forms, import: `ReactiveFormsModule`  

from `@angular/forms`.

---

### Steps to Create a Reactive Form

1. Import `ReactiveFormsModule`
2. Create a `FormGroup`
3. Define `FormControl` objects
4. Bind the form to the template
5. Handle form submission

Example:

```ts
import { FormGroup, FormControl } from '@angular/forms';

reactiveForm = new FormGroup({
  firstName: new FormControl(''),
  lastName: new FormControl('')
});
```

Template:

```html
<form [formGroup]="reactiveForm">

  <input formControlName="firstName">
  <input formControlName="lastName">

</form>
```

---

### Building Blocks of Reactive Forms

#### FormControl

Represents a **single input field**.

Example:

```ts
new FormControl('')
```

---

#### FormGroup

Represents a **collection of form controls**.

Example:

```ts
new FormGroup({
  name: new FormControl('')
})
```

---

#### FormArray

Represents a **dynamic list of form controls**.

Example:

```ts
new FormArray([])
```

Useful for fields like **skills, phone numbers, etc.**

---

### Reactive Form Example (Using FormBuilder)

Example using `FormBuilder`:

```ts
constructor(private fb: FormBuilder) {

  this.reactiveForm = this.fb.group({

    firstName: [ '', [Validators.required, Validators.pattern("^[a-zA-Z]+$")] ],
    lastName: [ '', [Validators.required, Validators.minLength(5)] ],
    email: [ '', [Validators.required, Validators.email] ],
    password: [ '', [Validators.required] ],
    isCheck: [],
    address: this.fb.group({
      city: [''],
      street: [''],
      pincode: []
    }),
    skills: this.fb.array([])
  });

  this.reactiveForm.controls['isCheck'].disable();

}
```

```html
<form [formGroup]="reactiveForm" (ngSubmit)="onSubmit()">

  <input formControlName="firstName">
  <input formControlName="lastName">
  <input formControlName="email">

  <button type="submit">Submit</button>

</form>
```

# Template-Driven vs Reactive Forms

| Feature        | Template Forms | Reactive Forms  |
| -------------- | -------------- | --------------- |
| Setup          | Simple         | More structured |
| Logic Location | HTML           | TypeScript      |
| Scalability    | Low            | High            |
| Testing        | Difficult      | Easier          |
| Use Case       | Small forms    | Complex forms   |
