# Angular Lifecycle Hooks

**Lifecycle Hooks** are a set of methods that Angular calls **at different stages of a component's lifecycle**.  

They allow developers to **run specific code when important events happen**, such as:
* When a component is created
* When data changes
* When the view is rendered
* When the component is destroyed

Each component instance goes through a **lifecycle**, and Angular provides hooks to interact with these stages.

The lifecycle **ends when the component is destroyed**.


# Order of Lifecycle Hooks

Angular calls lifecycle hooks in the following order:

| Hook                    | Purpose                        |
| ----------------------- | ------------------------------ |
| `ngOnChanges`           | Detect input property changes  |
| `ngOnInit`              | Initialize component           |
| `ngDoCheck`             | Custom change detection        |
| `ngAfterContentInit`    | Content projection initialized |
| `ngAfterContentChecked` | Content checked                |
| `ngAfterViewInit`       | View initialized               |
| `ngAfterViewChecked`    | View checked                   |
| `ngOnDestroy`           | Cleanup before destroy         |

## 1. ngOnChanges

`ngOnChanges` is called **when an input property changes**.  
It runs **before `ngOnInit`** and receives a `SimpleChanges` object containing previous and current values.  

Common use:
* Detect changes in `@Input()` properties  

Example:
```ts
import { Component, OnChanges, SimpleChanges } from '@angular/core';

ngOnChanges(changes: SimpleChanges): void {
  console.log("1. ngOnChanges called");
}
```
## 2. ngOnInit
`ngOnInit` is called **once after the component is initialized**.  

It is commonly used for:
* API calls
* Initializing data
* Setting up component logic

Example:
```ts
ngOnInit(): void {
  console.log("2. ngOnInit called");
}
```

## 3. ngDoCheck

`ngDoCheck` is called **every time Angular runs change detection**.  
It allows developers to **implement custom change detection logic**.  

Example:
```ts
ngDoCheck(): void {
  console.log("3. ngDoCheck called");
}
```

## 4. ngAfterContentInit
`ngAfterContentInit` is called **once after Angular projects external content into the component** using `ng-content`.  

Example:
```ts
ngAfterContentInit(): void {
  console.log("4. ngAfterContentInit called");
}
```

Common use:
* Work with projected content (`<ng-content>`)

## 5. ngAfterContentChecked
`ngAfterContentChecked` runs **after Angular checks the projected content**.  

Example:
```ts
ngAfterContentChecked(): void {
  console.log("5. ngAfterContentChecked called");
}
```

## 6. ngAfterViewInit
`ngAfterViewInit` is called **once after the component's view and child views are initialized**.  

Common use:
* Access `ViewChild` or `ViewChildren`
* Work with DOM elements

Example:
```ts
ngAfterViewInit(): void {
  console.log("6. ngAfterViewInit called");
}
```

## 7. ngAfterViewChecked

`ngAfterViewChecked` is called **after Angular checks the component’s view and child views**.  

Example:
```ts
ngAfterViewChecked(): void {
  console.log("7. ngAfterViewChecked called");
}
```

## 8. ngOnDestroy
`ngOnDestroy` is called **just before the component is destroyed**.  

Common use:
* Unsubscribe from Observables
* Remove event listeners
* Cleanup resources

Example:
```ts
ngOnDestroy(): void {
  console.log("8. ngOnDestroy called");
}
```

Example cleanup:

```ts
this.subscription.unsubscribe();
```
