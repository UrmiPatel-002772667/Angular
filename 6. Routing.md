# Angular Router

The **Router** in Angular is a module that enables **navigation between different components or views** in a single-page application (SPA).

Instead of loading a new page from the server, Angular **updates the view dynamically** based on the URL.

Angular routing helps to:

* Navigate between pages
* Maintain browser history
* Enable bookmarkable URLs
* Load components dynamically

### Why Angular Router is Used

Angular Router allows applications to:

* Create **Single Page Applications (SPA)**
* Map **URLs to components**
* Manage navigation without refreshing the page
* Implement **lazy loading**
* Protect routes using **guards**

## Setting Up Angular Router

To use routing, import the **RouterModule**.

Example:

```ts
import { RouterModule, Routes } from '@angular/router';
```

---

## Defining Routes

Routes define the **mapping between URL paths and components**.

Example:

```ts
const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent },
  { path: 'contact', component: ContactComponent }
];
```

Explanation:

| Path       | Component        |
| ---------- | ---------------- |
| `/`        | HomeComponent    |
| `/about`   | AboutComponent   |
| `/contact` | ContactComponent |

## Configuring RouterModule

Add routes to the application:

```ts
import { RouterModule } from '@angular/router';

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule {}
```

`forRoot()` configures the router at the **application root level**.

## Router Outlet

`<router-outlet>` is a **placeholder directive** where routed components are displayed.

Example:

```html
<router-outlet></router-outlet>
```

When the route changes, Angular **loads the corresponding component inside this outlet**.

---

## RouterLink Directive

`routerLink` is used to **navigate between routes** in templates.

Example:

```html
<a routerLink="/about">About</a>
<a routerLink="/contact">Contact</a>
```

This changes the URL **without reloading the page**.

---

## RouterLinkActive

`routerLinkActive` adds a CSS class when the route is active.

Example:

```html
<a routerLink="/about" routerLinkActive="active">About</a>
```

If the user is on `/about`, the `active` class is applied.

---

## Programmatic Navigation

Navigation can also be done inside a component using the **Router service**.

Example:

```ts
import { Router } from '@angular/router';

constructor(private router: Router) {}

goToHome() {
  this.router.navigate(['/home']);
}
```

---

## Route Parameters

Route parameters allow dynamic values in URLs.

Example route:

```ts
{ path: 'user/:id', component: UserComponent }
```

Example URL:

```
/user/10
```

Access parameter inside component:

```ts
import { ActivatedRoute } from '@angular/router';

constructor(private route: ActivatedRoute) {}

ngOnInit() {
  const id = this.route.snapshot.paramMap.get('id');
}
```

---

## Query Parameters

Query parameters provide additional data in the URL.

Example URL:

```
/products?category=electronics
```

Access query parameters:

```ts
this.route.queryParams.subscribe(params => {
  console.log(params['category']);
});
```

## Child Routes

Child routes allow nested routing.

Example:

```ts
const routes: Routes = [
  {
    path: 'dashboard',
    component: DashboardComponent,
    children: [
      { path: 'stats', component: StatsComponent },
      { path: 'reports', component: ReportsComponent }
    ]
  }
];
```

Example URLs:

```
/dashboard/stats
/dashboard/reports
```

---

## Lazy Loading

Lazy loading loads modules **only when needed**, improving performance.

Example:

```ts
{
  path: 'admin',
  loadChildren: () =>
    import('./admin/admin.module').then(m => m.AdminModule)
}
```

This loads the **Admin module only when the user visits `/admin`**.

---

## Route Guards

Route guards control **access to routes**.

Common guards:

| Guard         | Purpose                    |
| ------------- | -------------------------- |
| CanActivate   | Allow or deny route access |
| CanDeactivate | Prevent leaving a route    |
| Resolve       | Fetch data before loading  |
| CanLoad       | Prevent module loading     |

Example:

```ts
{ path: 'admin', component: AdminComponent, canActivate: [AuthGuard] }
```

---

## Wildcard Route

Wildcard routes handle **invalid URLs**.

Example:

```ts
{ path: '**', component: PageNotFoundComponent }
```

If the route does not match any path, Angular loads the **404 page**.

---

## Router Events

The Angular router emits events during navigation.

Example:

```ts
this.router.events.subscribe(event => {
  console.log(event);
});
```

Common events:

* NavigationStart
* NavigationEnd
* NavigationCancel
* NavigationError
