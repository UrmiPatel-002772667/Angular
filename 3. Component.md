# Angular Component

A **component** is the **main building block of an application in** Angular.

Each component represents a **specific part of the user interface (UI)**.
By dividing the application into multiple components, Angular applications become **modular, reusable, and easier to maintain**.

Organizing an application into components helps:

* Provide a clear project structure
* Separate logic and UI into manageable parts
* Improve maintainability and scalability

Modern Angular components are **standalone**, meaning they can be directly imported into other components without needing an NgModule.

---

# Basic Structure of a Component

Every Angular component contains three main parts:

1. **TypeScript Class**

   * Contains logic, properties, and methods
   * Handles user interactions and data processing

2. **HTML Template**

   * Defines what is displayed in the browser

3. **CSS Styles**

   * Controls the appearance of the component

4. **Selector**

   * Defines how the component is used inside HTML

---

# Example Component

```ts
import { Component } from '@angular/core';

@Component({
  selector: 'profile',
  template: `<h1>ABC</h1>`,
  styles: [`img { border-radius: 50%; }`]
})
export class ProfileComponent {}
```

Usage in another template:

```html
<profile></profile>
```

You can **compose multiple components together** by importing them:

```ts
@Component({
  imports: [ProfileComponent]
})
```

# Component Selectors

A **selector** determines how a component is used inside HTML.

Angular **matches selectors during compile time**.

Important rules:

* An element can match **only one component selector**
* Selectors are **case sensitive**
* DOM changes at runtime **do not affect component matching**

Example:

```html
<profile></profile>
<p>Profile component used</p>
```

---

# Types of Selectors

### Type Selector

Matches elements by their HTML tag name.

Example:

```
profile-photo
```

Usage:

```html
<profile-photo></profile-photo>
```

---

### Attribute Selector

Matches elements based on an **HTML attribute**.

Example:

```
[dropzone]
[type="reset"]
```

Usage:

```html
<div dropzone></div>
```

---

### Class Selector

Matches elements based on a **CSS class**.

Example:

```
.menu-item
```

Usage:

```html
<div class="menu-item"></div>
```

---

# :not() Pseudo-Class

The `:not()` selector excludes certain elements.

Example:

```ts
selector: '[dropzone]:not(textarea)'
```

This matches elements with `dropzone` **except textarea elements**.

---

# Combining Selectors

Selectors can be combined.

Example:

```ts
selector: 'button[type="reset"]'
```

Matches:

```
<button type="reset">
```

# Styling Components – Style Scoping

Angular components support **style encapsulation**, meaning styles can be limited to a specific component.

This is controlled using **ViewEncapsulation**.

Example:

```ts
encapsulation: ViewEncapsulation.None
```

---

# View Encapsulation Modes

## ViewEncapsulation.Emulated (Default)

Angular adds unique attributes to elements so styles apply **only to that component**.

Example:

```
_component-style scoping
```

---

## ViewEncapsulation.ShadowDom

Uses the browser's **Shadow DOM**.

* Styles apply only inside the shadow tree
* Outside elements cannot affect it

---

## ViewEncapsulation.ExperimentalIsolatedShadowDom

A stricter version of Shadow DOM.

* Component styles are completely isolated
* Global styles cannot affect the component

---

## ViewEncapsulation.None

Disables style encapsulation.

All styles behave as **global styles**.

---

# ::ng-deep

`::ng-deep` allows styles to **penetrate component boundaries**.

Example:

```
::ng-deep p a
```

This can style `<a>` elements **anywhere in the application**.

However, Angular **discourages using `::ng-deep`** and keeps it only for backward compatibility.

# Accepting Data with Input Properties

Components can accept data from parent components using **inputs**.

Angular provides the **`input()` function**.

Example:

```ts
import { Component, input, computed } from '@angular/core';

@Component({})
export class CustomSlider {

  value = input(0);

  label = computed(() => `The slider's value is ${this.value()}`);

}
```

---

# Required Inputs

Inputs can be marked as required.

```ts
value = input.required<number>();
```

Angular ensures that the parent component **must provide this value**.

---

# Configuring Inputs

Inputs can accept a **configuration object**.

Example:

```ts
widthPx = input('', { transform: appendPx });

function appendPx(value: number): string {
  return `${value}px`;
}
```

---

# Input Transforms

Transforms modify the value when Angular sets the input.

Common use cases:

* Convert values
* Handle null or undefined
* Accept multiple types

---

# Built-in Input Transformations

Angular provides:

* `booleanAttribute`
* `numberAttribute`

Example:

```ts
disabled = input(false, { transform: booleanAttribute });
```

---

# Model Inputs (Two-Way Binding)

Model inputs allow **two-way data binding** between parent and child components.

Example:

```
[(value)]
```

Model inputs allow the component to **update the parent value automatically**.

Angular automatically creates an output event called:

```
valueChange
```

---

# Custom Events with Outputs

Components can emit custom events using **outputs**.

Example:

```ts
this.valueChanged.emit(7);
```

Usage:

```html
<custom-slider (valueChanged)="logValue($event)"></custom-slider>
```

Here `$event` contains the emitted value.

---

# Customizing Output Names

Output names can be changed using an alias.

Example:

```ts
changed = output({ alias: 'valueChanged' });
```

Usage:

```html
<custom-slider (valueChanged)="saveVolume()"></custom-slider>
```

---

# Programmatically Subscribing to Outputs

When creating components dynamically:

```ts
const componentRef = viewContainerRef.createComponent(SomeComponent);

componentRef.instance.someEvent.subscribe(data => {
  console.log(data);
});
```

# Content Projection (ng-content)

Content projection allows a component to **display content provided by its parent**.

Example:

```ts
@Component({
  selector: 'custom-card',
  template: `
    <div class="card-shadow">
      <ng-content></ng-content>
    </div>
  `
})
export class CustomCard {}
```

Usage:

```html
<custom-card>
  <p>Hello World</p>
</custom-card>
```

---

# Multiple Content Placeholders

Angular allows multiple `<ng-content>` slots.

Example:

```html
<ng-content select="card-title"></ng-content>
<ng-content select="card-body"></ng-content>
```

---

# Fallback Content

Fallback content appears if no content is projected.

Example:

```html
<ng-content>Default Content</ng-content>
```

---

# ngProjectAs

`ngProjectAs` lets you **alias content for projection**.

Example:

```html
<div ngProjectAs="card-title">
  Title
</div>
```

# Host Elements

The element matching a component selector is called the **host element**.

Example:

```ts
@Component({
  selector: 'profile-photo',
  template: `<img src="profile.jpg">`
})
export class ProfilePhoto {}
```

Usage:

```html
<profile-photo></profile-photo>
```

Here `<profile-photo>` is the **host element**.

---

# Binding to Host Elements

Properties and events can be bound to the host element using the `host` property.

Example:

```ts
@Component({
  host: {
    'role': 'slider',
    '[attr.aria-valuenow]': 'value',
    '[class.active]': 'isActive()',
    '[style.background]': 'hasError() ? "red" : "green"',
    '[tabIndex]': 'disabled ? -1 : 0',
    '(keydown)': 'updateValue($event)'
  }
})
```

---

# @HostBinding

`@HostBinding` binds a property to the **host element**.

Example:

```ts
import { HostBinding } from '@angular/core';

@HostBinding('class.active')
isActive = true;
```

Result:

```
<component class="active"></component>
```

---

# @HostListener

`@HostListener` listens to **events on the host element**.

Example:

```ts
import { HostListener } from '@angular/core';

@HostListener('click')
onClick() {
  console.log('Clicked');
}
```

Example with event object:

```ts
@HostListener('keydown', ['$event'])
handleKey(event: KeyboardEvent) {
  console.log(event.key);
}
```
